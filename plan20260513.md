# brave-mcp-server — Plan 2026-05-13

## OG-meta fast-path az SM-platformok URL-jeire

### Probléma

A jelenlegi 7-szintű chain a Twitter/X status URL-en **timeoutol 45 mp-nél a
L5 FlareSolverr+render szinten**, mert a Twitter SPA túl komplex ahhoz, hogy
ennyi idő alatt teljes render-rel előálljon. Konkrét teszt:

```
URL: https://x.com/elonmusk/status/2052076315306864756
auto_fallback: true
escalation_path:
  L1 default       → js_required_stub
  L2 stealth       → js_required_stub
  L3 Webclaw       → empty_response
  L4 FlareSolverr  → js_required_stub
  L5 FS+render     → "Navigation timeout of 45000 ms exceeded"
  L6 Wayback       → empty
  L7 Google AMP    → ERR_ABORTED
Eredmény: content_usable=false, block_reason=all_levels_blocked
```

Idő: ~120 s. Eredmény: **nem idézhető tartalom.**

### Megoldás

Egy URL-pattern-felismerő **L0 OG-fast-path szint** a chain ELÉ, ami SM-poszt
URL-eknél (Twitter/X, Reddit, LinkedIn, Threads, Instagram, Facebook,
Mastodon, Bluesky, HackerNews) **nem indít headless browsert**, csak egy sima
HTTP GET-tel bot-UA-val kihúzza az `og:description` (vagy `twitter:description`)
meta-tagot.

A teljes poszt-szöveg ezekben a tagekben van, mert a link-preview kártyák
(Twitter Card, Discord-embed, Slack-preview, stb.) miatt a platformok
**szándékosan** ott hagyják őket még a JS-stub-on is.

### Bizonyíték

Echolot oldalon megcsináltuk (`echolot_og_fastpath.py`):

| Mód | Idő | Eredmény |
|---|---|---|
| Brave 7-szintű chain | ~120 s | `content_usable=false`, timeout, stub |
| OG-fast-path | **~560 ms** | `content_usable=true`, **teljes tweet text** |

Konkrét output a fenti Elon-tweetre:

```
@MobofJoggers @nottombrown Just as SpaceX launches hundreds of satellites
for competitors with fair terms and pricing, we will provide compute to AI
companies that are taking the right steps to ensure it is good for humanity.

We reserve the right to reclaim the compute if their AI engages in actions
that...
```

A `og:description`-ban 306 char, a teljes elérhető tweet-szöveg link-preview
formátumban.

### Mit kell beépíteni a brave-mcp-server-be

**Új szint a chain elejére: L0 OG-fastpath**

```javascript
// pseudokód
const PLATFORM_PATTERNS = [
  { regex: /^https?:\/\/(?:x|twitter)\.com\/[^/]+\/status\/\d+/i, name: 'twitter' },
  { regex: /^https?:\/\/(?:www\.)?reddit\.com\/r\/[^/]+\/comments\//i, name: 'reddit' },
  { regex: /^https?:\/\/(?:www\.)?linkedin\.com\/(?:posts|feed\/update)\//i, name: 'linkedin' },
  { regex: /^https?:\/\/(?:www\.)?threads\.(?:net|com)\/@[^/]+\/post\//i, name: 'threads' },
  { regex: /^https?:\/\/(?:www\.)?instagram\.com\/(?:p|reel|tv)\//i, name: 'instagram' },
  { regex: /^https?:\/\/(?:www\.)?facebook\.com\/[^/]+\/posts\//i, name: 'facebook' },
  { regex: /^https?:\/\/bsky\.app\/profile\/[^/]+\/post\//i, name: 'bluesky' },
  { regex: /^https?:\/\/[^/]+\/@[^/]+\/\d+/i, name: 'mastodon' },
  { regex: /^https?:\/\/news\.ycombinator\.com\/item\?id=/i, name: 'hackernews' },
];

async function ogFastpath(url) {
  const platform = matchPlatform(url);
  if (!platform) return null;

  const res = await fetch(url, {
    headers: {
      'User-Agent': 'Mozilla/5.0 (compatible; OG-link-preview-bot)',
      'Accept': 'text/html,*/*',
    },
    timeout: 10000,
  });
  if (!res.ok) return null;
  const html = await res.text();

  // Extract og: and twitter: meta tags (both attribute orders)
  const meta = extractOGMeta(html);
  const text = (meta['og:description'] || meta['twitter:description'] || '').trim();
  if (!text) return null;

  return {
    content_usable: true,
    block_reason: null,
    title: meta['og:title'] || meta['twitter:title'] || '',
    text,
    markdown: text,
    og_meta: meta,
    escalation_path: [{ level: 0, mode: 'og_fastpath', platform, ok: true, md_len: text.length }],
    cf_status: 'og_fastpath',
  };
}
```

**Integrációs pont** a meglévő `brave_scrape({url, auto_fallback: true})`
tool-ban: a `runEscalationChain()` legelején, az L1 default szint ELŐTT.

**Új flag** (opcionális): `og_fastpath: bool = true` (default ON). Ha
explicit `og_fastpath: false`, akkor kihagyja a fast-path-ot, és rögtön az
L1 default-ra megy.

### Miért érdemes a brave-mcp-server-be tenni

- Egyetlen kliens sem fogja külön implementálni
- A Bridge, Kimi, GPT, manuál MCP-call, jövőbeni új MCP-kliensek mind
  egyformán élvezik
- Az `escalation_path` field konzisztens módon mutatja, hogy ez a `og_fastpath`
  szinten oldódott meg
- A 7-szintű chain továbbra is rendelkezésre áll cikk-URL-ekre

### Mi maradjon a kliensoldalon

Semmi. Ha a brave-mcp-server natívan tudja, az Echolot törölheti az
`echolot_og_fastpath.py` modult és csak Brave-et hív. Most ideiglenes
duplikáció — amíg a brave-mcp-server-ben nincs.

### URL-pattern lista (induló)

A `PLATFORM_PATTERNS` regex-listát rendszeresen frissíteni kell. Új platform
bevezetésénél: 1) pattern match-ellenőrzés, 2) verify hogy az platform tényleg
og:description-be teszi a poszt-szöveget. Ha nem (pl. paywall mögötti hosszú
LinkedIn-cikk), akkor sima 7-szintű chain-en megy.

### Echolot-oldali referencia

A működő implementáció: `Tamas54/hirelemzoMCP` repo, `echolot_og_fastpath.py`
modul, használva a `scrape_url` MCP tool-ban. URL-platform-registry, `fetch_og()`
függvény, OG-tag regex-ekkel. Lényeg ~120 sor.

A teszt eredménye `fast_path: "og:twitter"` flag-gel a response-ban — ez fogja
mutatni majd a Brave-mcp-server-ben az `escalation_path[0].mode = "og_fastpath"`-t.
